# 生成安卓调试安装包 (APK) 指南

由于您在 Windows 环境下开发 React Native 应用，生成安卓安装包是最直接的调试方式。

## 1. 确保环境准备就绪
在生成安装包之前，请确保您的开发环境已配置好：
*   **JDK 17**: React Native 0.73+ 需要 JDK 17。
*   **Android SDK**: 确保安装了 Android SDK Platform 34 (或 33) 和 Android SDK Build-Tools。
*   **环境变量**: `ANDROID_HOME` 已设置指向 SDK 路径。

## 2. 方式一：直接运行到设备/模拟器 (推荐开发时使用)
这种方式支持热重载 (Fast Refresh)，修改代码后手机立即生效。
1.  **连接手机**: 使用 USB 连接安卓手机，开启「开发者模式」和「USB 调试」。
2.  **运行命令**: 在终端中运行以下命令：
    ```powershell
    npm run android
    ```
    或者
    ```powershell
    npx react-native run-android
    ```
3.  **结果**: 应用会自动安装到手机并启动，同时电脑端会启动 Metro 服务窗口。

## 3. 方式二：生成独立的调试版 APK (推荐分享给他人测试)
如果您需要发给同事安装，或者想在没有电脑连接的情况下测试：

1.  **进入安卓目录**:
    ```powershell
    cd android
    ```
2.  **清理构建 (可选，防止缓存问题)**:
    ```powershell
    ./gradlew clean
    ```
3.  **构建 APK**:
    ```powershell
    ./gradlew assembleDebug
    ```
    *(注意：第一次构建可能需要几分钟下载 Gradle 依赖)*

## 4. 获取 APK 文件
构建成功后，APK 文件位于以下路径：
`android/app/build/outputs/apk/debug/app-debug.apk`

您可以直接将这个 `.apk` 文件发送到手机上安装。
